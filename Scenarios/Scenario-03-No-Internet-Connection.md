# Scenario 3: Network Connectivity Troubleshooting

## Problem

A user reported being unable to access internet resources.

## Troubleshooting Steps

1. Opened Command Prompt.
2. Ran `ipconfig` to verify network configuration.
3. Verified that the device had received a valid IPv4 address.
4. Tested connectivity using ping commands.
5. Verified communication with external resources.

## Resolution

Confirmed that the system maintained a valid network configuration and internet connectivity.

## What I Learned

Tools such as `ipconfig` and `ping` are useful for quickly diagnosing basic network issues.
